# PRD — Автоматизация учёта производительности компании и расчёт рентабельности

## Основная цель

Создание управленческого дашборда с разноуровневыми метриками для оперативного принятия решений.

## Проблемы

1. Нет чёткого понимания о рентабельности работы каждого сотрудника, процесса, оказываемой услуги и клиента в компании.
2. Нет возможности оперативно проводить АВС-анализ каждого клиента и сотрудника и соответственно увеличивать эффективность компании.
3. Расчёт з/п в ручном режиме.

## Решение

1. Автоматизированный сбор данных по выработке каждого сотрудника.
2. Свод данных выработки каждого сотрудника за определённый период с сортировкой по услугам, клиентам, отработанным дням и другим параметрам.
3. Сведение стоимости услуг для клиентов и себестоимости выполнения операций → расчёт рентабельности услуг и клиентов.
4. Автоматический расчёт заработной платы на основе табеля посещаемости.

## Что уже сделано

1. Сбор данных каждого сотрудника на ежедневной основе в ручном режиме; данные обрабатывает менеджер.
2. Фиксация вход/выход каждого сотрудника системой СКУД (face ID).
3. Таблица выплаты зарплаты ведётся в ручном режиме.
4. Отслеживаются темпы обработки товаров каждым сотрудником в ручном режиме.
5. Ведётся ДДС с разбивкой по категориям и сведением в PnL.

## Что нужно сделать

### 1. Веб-приложение (Google Apps Script) с двумя ролями

#### 1.1. Админская часть (менеджер)
- Менеджер выдаёт задание сотруднику с указанием плана на текущую смену.
- Менеджер может переназначить невыполненное задание другому сотруднику.

#### 1.2. Клиентская часть (сотрудник)

1. Сотрудник получает задание с описанием.
2. Для начала работы нажимает **"Старт"**.
3. При необходимости перерыва — **"Пауза"**.
4. При полном завершении задания в рамках смены — **"Закончил задание полностью"**.
5. Если работа не завершена к концу смены — **"Закончил частично"** + вводит количество обработанного товара.
6. На следующий рабочий день сотрудник может продолжить незавершённое задание кнопкой **"Продолжить выполнение задания"**.
7. Если задание переназначено другому сотруднику — тот видит неоконченное задание и может начать его выполнять.

#### 1.3. Учёт количества и расчёты

- При нажатии **"Стоп"** и **"Закончил частично"** сотрудник вводит количество обработанного товара → данные попадают в таблицу учёта.
- Количество обработанного товара сопоставляется с оказанными услугами → рассчитывается итоговая стоимость работы.
- Рассчитывается рентабельность: стоимость работы минус себестоимость (время сотрудника + материалы).

#### 1.4. Табель посещаемости

- Время нахождения сотрудника на складе заносится в табель посещаемости.
- Сопоставляется с выгрузкой из СКУД.
- Вычисляется время простоя; при превышении нормы — инициируется разбирательство.

## Метрики для дашборда

| # | Метрика |
|---|---------|
| 1 | Даты начала и окончания периода |
| 2 | ФИО сотрудника (мультивыбор) |
| 3 | Выбор услуги / услуг |
| 4 | Выбор клиента / клиентов |
| 5 | Количество обработанного товара |
| 6 | Темпы обработки товара |
| 7 | Гистограмма распределения темпов обработки по сотрудникам |
| 8 | Время простоя |
| 9 | Лучший / худший сотрудник |
| 10 | Выручка с клиента за период |
| 11 | Выручка с сотрудника за период |
| 12 | Выручка с услуги за период |
| 13 | Блок рентабельности клиентов с АВС-анализом (лучший / худший клиент) |
| … | Другие метрики — в процессе добавления |
